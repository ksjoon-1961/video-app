# 화면 편집 가이드 — 버튼 · 색상 · 배치 · 장식 이미지 (초보자용)

화면을 꾸미는 작업은 대부분 **`static/css/style.css`** 한 파일에서 이뤄집니다.
데이터(영상 목록)나 로그인 같은 기능은 건드릴 필요가 없어 안전합니다.
이 문서는 "무엇을 / 어느 파일에서 / 어떻게" 바꾸는지와, Claude Code에
**말로 요청하는 예시 문장**까지 담았습니다.

---

## 0. 어디를 바꾸면 무엇이 바뀌나

| 바꾸고 싶은 것 | 만지는 파일 |
|---|---|
| 버튼 색·크기·모양·배치·간격 | `static/css/style.css` |
| 배경색 / 배경 이미지 | `static/css/style.css` |
| 로고·장식 이미지를 화면에 배치 | `templates/index.html` (+ 이미지 파일은 `static/img/`) |
| 설치된 앱 상단 표시줄 색 | `static/manifest.json`의 `theme_color` |
| 화면 문구·제목 | `templates/index.html` |

⚠️ **무엇을 바꾸든 끝나면** `static/sw.js`의 캐시 버전을 한 칸 올리세요
(`"v3"` → `"v4"`). 안 그러면 폰에 옛 화면이 캐시로 남습니다.

---

## 1. 두 가지 작업 방식

- **방식 A (추천)**: Claude Code에 **말로 요청**합니다. 예: "버튼을 더 둥글게,
  배경은 진한 남색, 글자는 흰색으로 바꿔줘." 초보자에게 가장 편합니다.
- **방식 B**: `style.css`에서 **색깔·숫자 값만 직접** 살짝 바꿔봅니다.
  구조를 건드리지 않고 값만 바꾸는 건 안전한 연습이 됩니다.

아래 설명에는 두 방식 모두에 쓸 수 있도록 **CSS 속성 이름**과
**요청 예시 문장**을 같이 적었습니다.

---

## 2. 버튼 꾸미기

### 색상
- 관련 속성: `background-color`(버튼 배경), `color`(글자색), `border`(테두리)
- 요청 예시: "영상 버튼 배경을 파란색(#2563EB), 글자는 흰색으로 해줘."

### 모서리 둥글기
- 관련 속성: `border-radius` (값이 클수록 둥글고, 아주 크면 알약 모양)
- 요청 예시: "버튼 모서리를 좀 더 둥글게(border-radius 16px) 해줘."

### 크기 · 글자 크기
- 관련 속성: `padding`(버튼 안 여백), `font-size`(글자 크기)
- 화면에 맞춰 자동으로 커지고 작아지게 하려면 `clamp()`를 씁니다(이미 적용됨).
- 요청 예시: "버튼 글자를 조금 키우고, 버튼 높이를 더 키워줘. 단 12개가 한
  화면에 들어오는 건 유지해줘."

### 한 줄에 몇 개 / 간격
- 관련 속성: `grid-template-columns`(열 수), `gap`(버튼 사이 간격)
- 지금은 세로 3열, 가로 4열입니다.
- 요청 예시: "세로로 볼 때 한 줄에 2개씩 보이게 하고, 버튼 간격을 넓혀줘."

### 누를 때 반응 효과
- 관련 속성: `:active`/`:hover` + `transition`
- 요청 예시: "버튼을 누르면 살짝 작아지는 효과를 넣어줘."

### "준비 중" 버튼 구분
- 미준비 버튼은 흐리게 보이게 되어 있습니다(투명도/채도 낮춤).
- 요청 예시: "준비 중 버튼은 회색으로, 자물쇠 느낌이 나게 흐리게 처리해줘."

---

## 3. 배경 꾸미기

### 단색 배경
- 관련 속성: 화면 전체(`body`)의 `background-color`
- 요청 예시: "전체 배경을 아주 연한 회색으로 바꿔줘."

### 배경 이미지 / 그라데이션
- 관련 속성: `background-image`, `background-size: cover`(화면 꽉 채움)
- 사진 위에 글자가 안 보일 수 있으니, **어두운 반투명 막(overlay)**을 한 겹
  깔아 가독성을 확보합니다.
- 요청 예시: "배경에 `/static/img/bg.jpg` 이미지를 화면에 꽉 차게 깔고,
  글자가 잘 보이도록 어둡게 살짝 덮어줘."

---

## 4. 장식 이미지(로고·배너 등) 넣기 — 구체 절차

**1단계 — 이미지 파일 준비해 폴더에 넣기**
프로젝트에 `static/img/` 폴더를 만들고 이미지를 넣습니다.
예: `static/img/logo.png`, `static/img/banner.jpg`

**2단계 — 화면에 표시하기 (두 가지 중 택1)**
- 화면 상단에 로고처럼 보이기 → `templates/index.html`의 적당한 위치에
  `<img src="/static/img/logo.png" alt="로고" class="logo">` 추가.
  (크기·위치는 `style.css`에서 `.logo` 로 조절)
- 화면 배경으로 깔기 → 3번의 "배경 이미지" 방식 사용.

**요청 예시**: "`static/img/logo.png`를 화면 맨 위 가운데에 너무 크지 않게
넣어줘. 버튼 영역은 그대로 한 화면에 유지해줘."

**이미지 고를 때 주의 (중요)**
- **용량**: 폰 데이터·속도에 직접 영향. 사진은 가능하면 수백 KB 이하로 압축.
- **형식**: 사진은 `jpg`, 로고·아이콘처럼 투명 배경이 필요하면 `png`,
  단순한 도형/패턴이면 `svg`(어떤 크기로도 깨지지 않음).
- **저작권**: 무료 사용 가능한 이미지(예: Unsplash, Pexels)나 직접 만든 것 사용.

---

## 5. 색 고르기 팁 (초보자)

- 메인 색을 **1~2개**만 정하면 깔끔합니다(예: 버튼색 + 강조색).
- **대비**를 충분히: 어두운 배경엔 밝은 글자, 밝은 배경엔 어두운 글자.
- 색 조합이 막막하면 무료 도구(예: coolors.co)에서 팔레트를 뽑아,
  그 색 코드(예: `#2563EB`)를 Claude Code에 그대로 알려주면 됩니다.
- 테마색을 바꿨다면 `manifest.json`의 `theme_color`도 같이 맞추면
  설치된 앱의 상단 색까지 통일됩니다.

---

## 6. Claude Code에 잘 요청하는 법

- **구체적으로**: "예쁘게 해줘"보다 "버튼 배경 #2563EB, 글자 흰색, 모서리
  둥글게, 한 줄에 2개"처럼 색·모양·개수를 콕 집어 말합니다.
- **한 번에 하나씩**: 바꾸고 폰에서 확인한 뒤 다음 걸 바꿉니다.
- **제약을 명시**: "12개 버튼이 스크롤 없이 한 화면에 들어오는 건 유지해줘"를
  꼭 덧붙이면 P5에서 맞춰둔 화면 최적화가 깨지지 않습니다.
- **되돌리기**: 마음에 안 들면 "방금 디자인 변경 되돌려줘"라고 하면 됩니다.

---

## 7. 바꾼 뒤 폰에 반영하는 순서

1. (Claude Code로) `style.css` 등 수정.
2. `static/sw.js` 캐시 버전 한 칸 올리기.
3. `pytest` 통과 확인 → `git push` → Railway 자동 배포(몇 분).
4. 폰에서 새로고침. 아이콘을 바꿨다면 기존 아이콘 삭제 후 재설치.

---

## 8. 흔한 함정 체크리스트

- [ ] 디자인 바꾸고 **`sw.js` 캐시 버전을 안 올림** → 폰에 옛 화면이 그대로.
- [ ] 이미지 파일이 **너무 큼** → 로딩이 느려짐(압축 필요).
- [ ] **글자색과 배경색 대비가 약함** → 글씨가 잘 안 보임.
- [ ] 디자인을 바꾸다 **12개 버튼이 한 화면을 넘어 스크롤이 생김**
      → 요청 시 "한 화면 유지" 제약을 명시.
- [ ] 이미지 **경로 오타**(`/static/img/...`) → 이미지가 안 뜸.

---

## 9. 안전한 첫걸음 (직접 해보기)

`static/css/style.css`에서 영상 버튼의 `background-color` 값 하나만
원하는 색 코드로 바꿔보세요. 그다음 `sw.js` 버전을 올리고 배포해 폰에서
색이 바뀌는지 확인하면, 화면 편집의 전체 흐름을 한 번에 익힐 수 있습니다.
익숙해지면 색 → 모양 → 배치 → 이미지 순으로 넓혀가시면 됩니다.
