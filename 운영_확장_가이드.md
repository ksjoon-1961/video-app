# 비디오 라이브러리 앱 — 운영 · 확장 가이드 (초보자용)

이 문서 하나로 ① 지금까지 무엇을 만들었는지 ② 동영상을 추가하는 법
③ 디자인을 바꾸는 법 ④ 새 기능을 추가하는 법 ⑤ 자주 막히는 함정을
모두 확인할 수 있습니다. 코드를 잘 몰라도 따라 할 수 있게 정리했습니다.

---

## 1. 이 앱이 하는 일

가입·로그인한 사용자에게 영상 이름이 적힌 **버튼들**을 보여주고, 버튼을
누르면 라이브러리에 등록된 **동영상을 재생**합니다. 안드로이드 폰에서
"홈 화면에 추가"하면 **설치한 앱처럼** 전체화면으로 실행됩니다.

---

## 2. 어떤 기술로 어떻게 동작하나 (큰 그림)

- **화면을 보여주는 곳(백엔드/서버)**: FastAPI (파이썬). Railway에 올라가 있어
  인터넷 주소(URL)로 접속됩니다.
- **로그인/회원 관리**: Supabase Auth가 대신 처리합니다. 비밀번호 저장 같은
  위험한 일은 우리가 직접 하지 않습니다.
- **영상 목록 데이터**: Supabase의 데이터베이스 표(`videos` 테이블)에 저장됩니다.
- **영상 파일**: Supabase Storage의 **비공개(Private)** `videos` 버킷에 보관됩니다.
  로그인한 사람에게만 잠깐 열리는 **임시 링크(signed URL)**로 재생됩니다.

**한 줄 요약 흐름**: 로그인 → 서버가 영상 목록을 내려줌 → 버튼 클릭 →
서버가 그 영상의 임시 재생 링크를 만들어 줌 → 폰에서 재생.

> 보안 원칙: 서버는 항상 "로그인한 사용자인지" 확인한 뒤에만 목록과 재생
> 링크를 내줍니다. 비밀 키(secret)는 쓰지 않고, 공개해도 되는 키만 사용합니다.

---

## 3. 지금까지 진행한 단계 (P0~P4)

| 단계 | 한 일 | 결과 |
|---|---|---|
| **P0** | 빈 골격 + 서버 + Railway 배포 검증 | 인터넷에서 접속되는 빈 앱 |
| **P1** | Supabase 로그인/회원가입 연결 | 가입·로그인 동작 |
| **P2** | 영상 목록 표 + 이름 버튼 표시 | 로그인 후 버튼들이 보임 |
| **P3** | 버튼 클릭 → 임시 링크 → 영상 재생 | 영상이 실제로 재생됨 |
| **P4** | PWA화 (아이콘·설치) | "홈 화면에 추가"로 앱처럼 설치 |

여기까지가 **MVP(최소 완성본)**입니다. 앞으로는 이 위에 동영상을 더하고,
디자인을 바꾸고, 기능을 추가하면 됩니다.

## 4. 폴더 구조와 각 파일의 역할

```
videoapp/
├── PROJECT.md            # 전체 설계·원칙 (바뀌면 안 되는 기준)
├── PROMPT.md             # '지금 작업'을 적는 지시서 (단계마다 내용 교체)
├── app/
│   ├── main.py           # 서버 시작점
│   ├── config.py         # 환경변수(주소·키 등) 읽기
│   ├── auth.py           # 로그인 토큰 검증
│   ├── routers/
│   │   ├── pages.py      # 화면(HTML) + /sw.js 제공
│   │   └── api.py        # /api/videos, /api/videos/{id}/url 등 데이터 제공
│   └── services/
│       ├── catalog.py    # videos 표 조회
│       └── storage.py    # 임시 재생 링크(signed URL) 생성
├── static/
│   ├── manifest.json     # 앱 이름·아이콘·색상 (PWA 설정)
│   ├── sw.js             # 설치/캐시 담당 (디자인 바꾸면 버전 올리기!)
│   ├── css/style.css     # ★ 디자인(색·버튼·배치)은 주로 여기
│   ├── js/app.js         # 버튼 렌더링 + 클릭 시 재생
│   └── icons/            # 앱 아이콘 192/512
└── templates/
    ├── login.html        # 로그인 화면
    └── index.html        # ★ 메인 화면(버튼 + 영상 플레이어)
```

★ 표시가 **디자인을 바꿀 때 주로 만지는 파일**입니다.

## 5. 자주 하는 작업 가이드

### 5-1. 동영상 추가하기 (가장 자주 할 작업)

**1단계 — 영상 파일 올리기**
Supabase 대시보드 → Storage → `videos` 버킷 클릭 → **Upload file**로 mp4 업로드.
올린 파일 이름을 그대로 메모합니다(예: `Lesson-04.mp4`). 대소문자까지 정확히.

**2단계 — 목록 표에 한 줄 추가**
Supabase → SQL Editor에서 실행:
```sql
insert into public.videos (name, storage_path, sort_order)
values ('4번째 영상 제목', 'Lesson-04.mp4', 4);
```
- `name`: 버튼에 보일 이름
- `storage_path`: **버킷명(videos)은 빼고** 올린 파일 이름만. 대소문자 일치 필수.
- `sort_order`: 버튼 정렬 순서(숫자가 작을수록 위).

끝입니다. 새로고침하면 새 버튼이 나타나고 클릭하면 재생됩니다. 코드는 건드릴
필요 없습니다.

### 5-2. 동영상 수정 / 삭제

**이름·순서 바꾸기**
```sql
update public.videos set name = '새 제목', sort_order = 2 where id = '해당-영상-id';
```
**삭제하기** (두 곳 모두 지워야 깔끔)
```sql
delete from public.videos where id = '해당-영상-id';
```
그리고 Storage `videos` 버킷에서 해당 mp4 파일도 삭제합니다.

> 영상 id는 SQL Editor에서 `select id, name from public.videos;` 로 확인합니다.

### 5-3. 디자인 바꾸기

대부분 코드 한두 파일만 바꾸면 됩니다. Claude Code에 "이렇게 바꿔줘"라고
요청할 때 아래를 참고하세요.

- **색상 테마 바꾸기**: `static/css/style.css`의 색상값. 단, 앱 테마색을 바꾸면
  `static/manifest.json`의 `"theme_color"`와 `index.html`의 `theme-color`도 같이.
- **버튼 모양·크기·간격**: `static/css/style.css`.
- **화면 구성/문구**: `templates/index.html`(메인), `templates/login.html`(로그인).
- **앱 아이콘 교체**: `static/icons/`의 png 두 개를 같은 이름으로 덮어쓰기.
  (아이콘은 가운데에 콘텐츠를 모으고 배경을 끝까지 채워야 안 잘립니다.)

⚠️ **디자인/아이콘/정적 파일을 바꾼 뒤에는 반드시** `static/sw.js`의 캐시 버전을
올리세요(`"v1"` → `"v2"`). 안 그러면 사용자 폰에 옛 화면이 캐시로 남습니다.

### 5-4. 새 기능 추가하기 (지금까지의 방식 그대로)

새 기능도 P0~P4와 똑같은 **단계(페이즈) 방식**으로 추가하면 안전합니다.

1. `PROJECT.md`(전체 원칙)를 먼저 확인.
2. 새 `PROMPT.md`를 작성: **목표 / 범위(할 것·안 할 것) / 산출물(파일별) /
   바뀌면 안 되는 것(Do Not Break) / 완료 기준** 을 적습니다.
3. Claude Code에 그 `PROMPT.md`를 지시.
4. 완료 기준(특히 기존 테스트 전부 통과)을 확인한 뒤 다음으로 진행.

> 이 문서 작성자(저)에게 "○○ 기능 추가용 PROMPT.md 만들어줘"라고 하시면,
> 이 형식에 맞춰 지시서를 만들어 드릴 수 있습니다.

## 6. 코드를 바꾼 뒤 실제 폰에 반영되는 순서

1. (Claude Code로) 코드 수정.
2. `pytest` 전체 통과 확인.
3. `git add . → git commit → git push` (GitHub: `ksjoon-1961/video-app`).
4. **Railway가 자동으로 새 버전을 배포**합니다(몇 분 소요).
5. 정적 파일(css/js/아이콘)을 바꿨다면 `sw.js` 캐시 버전을 올렸는지 확인.
6. 폰에서 새로고침해 확인. 아이콘을 바꿨다면 **기존 아이콘 삭제 후 재설치**.

## 7. 자주 막히는 함정 체크리스트

- [ ] `videos` 버킷은 **반드시 Private(비공개)**. Public으로 만들면 누구나
      URL로 영상에 접근하게 됩니다.
- [ ] 영상 추가 시 표의 `storage_path`와 실제 파일 이름의 **대소문자가 일치**
      해야 합니다(`Sample-1.mp4` ≠ `sample-1.mp4`).
- [ ] Supabase의 **RLS 정책**이 없으면 목록이 비거나 재생이 안 됩니다.
      (videos 표 읽기 정책 + storage.objects 읽기 정책 둘 다 필요)
- [ ] 정적 파일을 바꾸면 `sw.js`의 **캐시 버전을 올려야** 새 내용이 반영됩니다.
- [ ] service worker는 `/sw.js`(**루트 주소**)에서 제공되어야 앱 전체를 제어합니다.
- [ ] **비밀값(.env, secret 키)은 절대 GitHub에 커밋하지 않습니다.**
- [ ] 임시 재생 링크 만료시간(`SIGNED_URL_TTL`)은 영상 길이보다 넉넉하게.

## 8. 환경변수(주소·키) 관리

- 로컬 개발용은 `.env` 파일에, 운영용은 **Railway 대시보드 → Variables**에 넣습니다.
- 두 곳 모두 같은 값을 넣어야 로컬과 실제 서버가 똑같이 동작합니다.
- 공개돼도 되는 값: `SUPABASE_PUBLISHABLE_KEY`(프론트 노출 허용).
- 절대 노출 금지: secret 계열 키(현재 구조에서는 사용하지 않음).

주요 변수: `SUPABASE_URL`, `SUPABASE_JWKS_URL`, `SUPABASE_PUBLISHABLE_KEY`,
`SUPABASE_JWT_AUDIENCE`(기본 `authenticated`), `VIDEO_BUCKET`(기본 `videos`),
`SIGNED_URL_TTL`(초, 기본 3600).

## 9. 다음에 해볼 만한 일 (확장 아이디어)

- **맞춤 "앱 설치" 버튼**: 설치 안 한 사용자에게만 화면에 설치 버튼을 띄우고,
  설치 후엔 자동으로 숨기기(`beforeinstallprompt` 활용).
- **영상 등록 관리 화면**: SQL 없이 화면에서 영상 이름·파일을 추가/삭제.
- **이메일 인증 켜기**: 운영 단계에서 가입 시 이메일 확인 다시 활성화.
- **썸네일·카테고리·페이지네이션**: 영상이 많아질 때 보기 편하게.
- **사용자별 권한 분리**: 특정 사용자/그룹만 특정 영상을 보게 하려면
  `videos` 표와 RLS 정책에 소유자/그룹 컬럼 추가.

각 항목이 필요해지면, 그때 해당 기능용 `PROMPT.md`를 위 5-4 방식으로 만들어
한 단계씩 추가하면 됩니다.

## 10. 빠른 참조

**엔드포인트**
- `GET /health` — 서버 상태 점검(로그인 불필요)
- `GET /api/me` — 로그인한 본인 정보(로그인 필요)
- `GET /api/videos` — 영상 목록(로그인 필요)
- `GET /api/videos/{id}/url` — 임시 재생 링크(로그인 필요)
- `GET /sw.js` — service worker

**videos 테이블 컬럼**
`id`(자동), `name`(버튼 이름), `storage_path`(버킷 내 파일명),
`sort_order`(정렬), `created_at`(자동)

**필수 Supabase 설정**
- Auth: 비대칭(ES256) signing key 활성
- Storage: `videos` 버킷 = Private
- RLS: videos 읽기 정책 + storage.objects 읽기 정책(bucket_id = 'videos')
